# Writing Style Guide

> Rules for writing natural LinkedIn post content.

1. Use simple, clear language.
2. Write as you would speak
3. Make it so readers want to read the next lines.


## Avoid robotic lists

```diff
- Enterprise. Banks. Legacy systems.
+ You know, the whole enterprise world.
+ Banks, big corporations, all that.
```

```diff
- No transpilation step.
- No disappearing types.
- No runtime surprises.
+ Your types do not disappear.
+ Your code does not get rewritten.
+ What you write is what runs.
```

## No use of "?" for rhetorical pauses

```diff
- But today?
+ But today...
```

## Add relevant hashtag at the end

```diff
  That is learning.
  Everything else is entertainment.
+
+ #learningbydoing
```

## Optional emoji at the end (for dwell time)

```diff
- Everything else is entertainment.
+ Everything else is entertainment 🙂.
```

Makes readers want to finish the post. Good for engagement.

## Use emoji bullets for lists (sometimes)

```diff
- Discord rebuilt their Read States service in Rust.
- Figma uses Rust for their multiplayer sync engine.
+ 🔸 Discord rebuilt their Read States service in Rust.
+ 🔸 Figma uses Rust for their multiplayer sync engine.
```

```diff
- undefined is not a function
- Cannot read property of null
+ → undefined is not a function
+ → Cannot read property of null
```

```diff
- undefined is not a function
- Cannot read property of null
+ ⚠️ undefined is not a function
```

That's the mirror effect taken to its extreme. Good writers use 4 specific techniques for it:

1. Hyper-specific internal monologue
Don't describe what the reader does — describe what they think at 11pm when no one's watching.
Instead of "You tried to learn Rust" → "You had 14 tabs open. The Rust Book in one of them. You read the same paragraph three times and couldn't tell
anyone what it said."
The reader recognizes the 14 tabs. That's them.

2. Cinematic scenes — put them in a moment
Don't say "you felt frustrated." Put them in the room.
"You're in a standup. The backend team is explaining, again, why the APIs aren't ready. You're nodding. You've been nodding for three weeks."
They can see it. They've been in that room.

3. The secret they've never said out loud
The most powerful mirror is when you name the thought they've had but never admitted to anyone — including themselves.
"You didn't quit because Rust was hard. You quit because you couldn't tell if you were making progress, and you'd rather believe it's not for you than
find out it is and still fail."
That hits different. They've never heard it said that clearly.

4. Raw language — drop the polish
"Shit, the compiler is angry again" reads like a human wrote it.
"You may encounter compiler errors" reads like a manual.
The rawness signals: this person lived it. They didn't research it.

5. Specificity over adjectives
Numbers and details create credibility. Vague = marketing. Specific = lived experience.
"after a few weeks" → "after 3 weeks of nodding in that standup"
"it was slow" → "GC pauses spiking to 200ms"
"many tabs open" → "14 tabs open"
The reader doesn't recognize "frustrated." They recognize "14 tabs."

6. The reframe — turn their liability into their edge
Take what they think disqualifies them and flip it into an advantage.
"Your PHP instincts aren't wasted. They're your unfair advantage."
"You don't start at zero. You start at PHP with a map."
Different from the mirror effect — this moves them forward instead of just making them feel seen.

7. Sentence rhythm — short punch after long setup
One long sentence builds tension. One short sentence releases it.
"You've been saying this for two years. Things will never slow down."
"The compiler rejects AI's first draft too. Often the second."
"You quit at the wrong moment. Try again."
If everything is short, nothing lands. The short line only hits because of what came before it.
